How this app works

Products are cleaned into tags, TF-IDF builds keyword similarity, semantic embeddings capture meaning-level similarity, a preference-profile vector is built from liked seed items, and popularity acts as a light stabilizer. These four signals are merged into one final hybrid ranking.

How to read this page

This homepage presents a default sample catalog from the dataset rather than personalized recommendation output. Items are ranked with rating and review-backed popularity so the interface remains useful before any preference input is provided.

Detailed recommendation signals such as `BestSignal`, `Keyword`, `Semantic`, `Profile`, and `Popularity` are shown on the recommendation page after a seed item and optional liked items are submitted.
